GrapesJS MJML PoC

Preserve the rendered <img> node during MJML updates

Both editors run the same grapesjs-mjml@1.0.8 package and the same MJML input. The patched editor applies an mj-image-only subtree reconciliation step.

Waiting

Each update toggles width="240px" ↔ width="420px" and cache-busts the image src to force a real reload. The baseline recreates the <img> (it blanks, then re-fetches); the patched editor keeps the same <img> node, so the picture swaps in place without flickering. A replacement is detected by comparing the rendered <img> DOM object before and after each update.
